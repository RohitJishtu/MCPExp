What Different components of Agent DO 


Component                | What it Does                                   | Why it Matters
-------------------------|-----------------------------------------------|-----------------------------
LLM                      | Generates text, reasons, uses trained knowledge| Provides intelligence, not actions
Persona                  | Defines role, tone, and behavior               | Aligns agent to business or user needs
Reasoning & Decision Making | Plans tasks, decomposes goals, reflects      | Makes the system goal-driven, not reactive
Action Execution         | Calls tools, APIs, and external systems        | Enables the agent to actually do work
Memory Management        | Stores context, experience, preferences        | Allows learning and personalization
AI Agent (Orchestrator)  | Coordinates LLM, memory, tools, and decisions  | Turns components into a working system

| API Type      | Full Form                       | Primary User        | Basic Uniqueness                             | Example                        |
| ------------- | ------------------------------- | ------------------- | -------------------------------------------- | ------------------------------ |
| **REST**      | Representational State Transfer | Humans & services   | Resource-based, stateless                    | `GET /users/123`               |
| **GraphQL**   | Graph Query Language            | Frontends           | Client controls data shape                   | `user { orders { total } }`    |
| **gRPC**      | Google Remote Procedure Call    | Internal services   | Binary, fast, strongly typed                 | `GetUser()`                    |
| **WebSocket** | WebSocket Protocol              | Real-time clients   | Persistent two-way connection                | Live chat                      |
| **Pub/Sub**   | Publish–Subscribe               | Systems             | Asynchronous, event-driven                   | Kafka event                    |
| **Tool APIs** | Function / Tool Calling APIs    | LLMs                | Lets models execute functions                | `get_weather(city)`            |
| **MCP**       | **Model Context Protocol**      | **AI Agents**       | **Intent-based, governed tool access**       | `get_table_schema(table)`      |
| **A2A**       | **Agent-to-Agent**              | **Agents ↔ Agents** | **Coordination & delegation between agents** | Impact agent ↔ Explainer agent |
